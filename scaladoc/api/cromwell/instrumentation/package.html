  1. object Instrumentation

    Bit of a hack.

    Bit of a hack. It's not always easy to simply wrap kamon calls with an if/else (and even if it were, it'd get messy quickly). Instead we're setting up a single global monitoring object which is either a simple wrapper around kamon or a noop construct.

    Note that not all of kamon is wrapped here, just the bits of instrumentation that we're using. If one adds new instrumentation it might require adding stuff here
